# Superpoder
Transforma "código que funciona" em "código limpo" usando princípios Clean Code + SOLID + padrões de projeto.

# Trigger
- "refactor this"
- "clean code"
- "improve architecture"
- "make it maintainable"

# Princípios Aplicados
1. **Single Responsibility** - Uma classe, uma responsabilidade
2. **Open/Closed** - Aberto para extensão, fechado para modificação
3. **Liskov Substitution** - Substituível sem quebrar
4. **Interface Segregation** - Interfaces específicas
5. **Dependency Inversion** - Dependa de abstrações

# Output
- Antes/Depois comparativo
- Explicação de cada mudança
- Benefícios (manutenibilidade, testabilidade, performance)
