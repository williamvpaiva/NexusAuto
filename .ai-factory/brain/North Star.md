# North Star - NexusAuto

## Visão
O NexusAuto é um orquestrador autônomo de desenvolvimento de software baseado em agentes de IA que operam sobre arquivos Markdown, evoluindo para um sistema com **cognição evolutiva**, **memória de longo prazo** e **otimização de custos**.

## Missão
Transformar o desenvolvimento de software em um pipeline contínuo de entrega de valor, onde cada agente é uma etapa especializada que encadeia sua entrega para gerar produto final, sem retrabalho e com aprendizado acumulativo.

## Objetivo Central
- **Memória Persistente:** Nenhuma sessão começa do zero. Todo conhecimento é armazenado e recuperável via busca semântica.
- **Economia de Tokens:** Cada token economizado é um passo em direção à escalabilidade. Cache, sumarização e contexto em camadas são obrigatórios.
- **Habilidades Padronizadas:** Skills com formato SKILL.md permitem descoberta, reuso e integração com ecossistemas externos.
- **Integração com Ferramentas:** Agentes podem executar ações reais (GitHub, Slack, Jira) via Composio.

## Princípios Fundamentais
1. **Pipeline, não lista de tarefas** - O valor está no encadeamento correto das entregas
2. **Memória é aprendizado** - Sem memória, não há evolução
3. **Simplicidade operacional** - Cada componente deve justificar sua complexidade
4. **Baixo custo por design** - Otimização de tokens não é opcional
5. **Padronização permite escala** - Skills e handoffs padronizados permitem crescimento

## Métricas de Sucesso
- Redução de 60%+ no consumo de tokens entre sessões (via cache e busca semântica)
- Recuperação de contexto relevante em <2 segundos
- Zero sessões começando do zero (100% de contexto carregado via SessionStart Hook)
- 10+ skills padronizadas criadas e utilizadas em produção
