# AI Factory Skills

## Visão Geral

Este diretório contém skills especializadas para a AI Factory Polymarketing. Cada skill fornece capacidades específicas para diferentes aspectos do desenvolvimento de software.

## Skills Disponíveis

### 📋 Skills de Engenharia de Requisitos

#### requirements-engineer
**Propósito:** Transformar demandas vagas em requisitos estruturados

**Quando usar:**
- Criar histórias de usuário com critérios de aceitação
- Definir requisitos funcionais e não-funcionais
- Priorizar features com MoSCoW ou Value vs Effort
- Escopar MVP e criar matrizes de rastreabilidade

**Principais capacidades:**
- Elicitação de requisitos (5 Whys, User Journey, Jobs to Be Done)
- User stories no formato "As um... quero... para..."
- Critérios de aceitação Given-When-Then
- Definição de NFRs (performance, segurança, escalabilidade)

**Arquivo:** `requirements-engineer/SKILL.md`

**Propósito:** Criar arquitetura de sistemas escaláveis

**Quando usar:**

**Arquivo:** `api-designer/SKILL.md`

## Como Usar Skills

### Em Qualquer IDE

1. **Identifique a tarefa:** Qual capacidade você precisa?
2. **Selecione a skill:** Escolha a skill apropriada
3. **Ative a skill:** Leia o SKILL.md antes de começar
4. **Siga o workflow:** Cada skill tem um workflow definido
5. **Verifique qualidade:** Use os quality standards da skill

### Combinação de Skills

Skills podem ser combinadas para tarefas complexas:

```
Exemplo: Nova Feature Completa

1. requirements-engineer → User stories e critérios
2. architecture-designer → Design da solução
3. api-designer → Contrato de API
4. test-strategist → Estratégia de testes
5. devops-automation → CI/CD pipeline
6. documentation-writer → Documentação
```

### Criando Novas Skills

Para criar uma nova skill:

1. Crie diretório: `mkdir nova-skill/{references,examples,scripts}`
2. Crie SKILL.md seguindo o template padrão
3. Documente: propósito, quando usar, capacidades, workflow
4. Adicione exemplos práticos
5. Atualize este INDEX.md

## Template de Skill

```markdown
# skill-name

## Purpose
{Descrição em 1-2 frases}

## When to Use This Skill
{Lista de cenários de uso}

## Core Capabilities
1. **Capability 1** - Description
2. **Capability 2** - Description
3. **Capability 3** - Description

## Workflow
{Passo a passo de como usar a skill}

## Quality Standards
{Critérios de qualidade}

## Error Handling
{Como a skill lida com erros}

## References
{Links para documentação}

## Examples
{Exemplos práticos}
```
